# 哆啦D梦的口袋：项目说明书

> 版本：1.0  
> 状态：已确认，作为第一版开发与部署基线  
> 最后更新：2026-07-10

## 1. 项目概述

### 1.1 名称

**哆啦D梦的口袋**

### 1.2 定位

一个以摄影内容为主、个人文章为辅的自托管个人网站。网站用于展示个人拍摄的照片、整理相册，以及发布个人文章。

网站的体验目标是：打开后像进入一个明亮、安静、装着照片和文字的小口袋；以大图、留白和内容本身为主，不堆砌互动模块或复杂功能。

### 1.3 第一版原则

- 以照片和文字展示为核心。
- 使用网页后台管理内容，不要求通过代码或 Markdown 发布。
- 不接入腾讯云 COS、CDN、第三方 CMS 托管或外部数据库服务。
- 所有网站数据保留在自有服务器与本地备份中，便于整体迁移。
- 保持轻量、稳定、易维护；不实现暂时不需要的功能。

## 2. 已确认的技术架构

```text
访问者浏览器
      |
      v
腾讯云轻量服务器
      |
      v
Docker Compose
  ├── Caddy
  |     ├── 对外提供域名访问
  |     ├── 自动申请与续期 HTTPS 证书
  |     └── 反向代理到 WordPress
  |
  ├── WordPress
  |     ├── 前台网站
  |     ├── /wp-admin 内容管理后台
  |     ├── 文章、相册、页面与媒体库
  |     └── 自定义主题
  |
  └── MariaDB
        └── 存储文章、相册、标签、设置和媒体记录

宿主机持久化目录
  ├── WordPress 上传图片
  ├── MariaDB 数据库数据
  ├── Caddy HTTPS 证书数据
  └── 配置与备份文件
```

### 2.1 技术选型

| 层级 | 技术 | 责任 |
| --- | --- | --- |
| CMS 与前台 | WordPress | 提供前台网站、`/wp-admin` 后台、文章编辑、媒体库、页面、标签和相册内容管理。 |
| 数据库 | MariaDB | 保存文章、相册、标签、网站设置、后台账号、媒体文件的描述与关联关系。 |
| Web 入口 | Caddy | 处理域名访问、HTTPS 证书、HTTP 自动跳转 HTTPS，并将请求转给 WordPress。 |
| 容器编排 | Docker Compose | 一次管理 Caddy、WordPress 和 MariaDB 三个服务。 |
| 主题 | 自定义 WordPress 主题 | 实现本站的视觉风格、首页、摄影、相册、文章、归档和关于页。 |
| 图片存储 | 服务器本地磁盘 | 存储 WordPress 上传的公开展示照片及 WordPress 自动生成的缩略图。 |
| 版本管理 | Git | 保存 Docker 配置、自定义主题、必要插件配置、项目文档与部署脚本。 |

### 2.2 明确不使用的技术

- 不使用腾讯云 COS。
- 不使用 CDN。
- 不使用 Nginx。
- 不使用 Certbot（由 Caddy 自动处理 HTTPS）。
- 不使用 Nuxt、Astro、Vue、React 作为本站前台主框架。
- 不使用 Headless CMS 或前后台分离架构。
- 不使用第三方 CMS 托管服务。
- 不在第一版引入评论、会员、表单、商城、私密相册或复杂搜索。

### 2.3 选择 WordPress 的原因

WordPress 作为一体化 CMS，已经提供：

- 后台写文章与保存草稿；
- 媒体库批量上传照片；
- 页面、标签、发布时间和文章修订历史；
- 登录管理；
- 主题机制与可扩展的内容类型。

因此不需要自行开发后台、上传接口、图片管理、内容编辑器、用户权限或数据库管理界面。自定义开发集中在主题与摄影相册体验上。

## 3. 内容与信息架构

### 3.1 前台主导航

```text
首页 | 摄影 | 文章 | 归档 | 关于
```

不设置友情链接、器材页、联系页或常规页脚导航。

### 3.2 页面清单

| 页面 | 目的 | 主要内容 |
| --- | --- | --- |
| 首页 | 快速传达网站气质与最新内容 | 精选大图、站点名称、简介、最新相册、最新文章。 |
| 摄影 | 展示所有相册 | 以年份为主分组展示相册封面。 |
| 相册详情 | 沉浸式查看一组照片 | 相册标题、年份、可选说明、照片网格、灯箱浏览。 |
| 文章 | 浏览所有文章 | 按发布时间倒序展示文章列表。 |
| 文章详情 | 阅读单篇文章 | 标题、日期、标签、正文、文中图片。 |
| 标签页 | 按标签浏览文章 | 文章标签列表与某标签下的文章集合。 |
| 归档 | 以时间线浏览全部内容 | 按年份汇总相册与文章。 |
| 关于 | 说明网站与作者 | 一段简短介绍文字。 |
| 404 | 处理不存在的地址 | 简洁提示与回首页入口。 |

### 3.3 首页结构

首页以摄影内容为绝对主角，文章为次级入口。

```text
首屏
├── 一张精选横向大图
├── 网站名称：哆啦D梦的口袋
├── 简短介绍文案
└── 两个入口：去看照片 / 读点文字

内容区
├── 最新相册（优先展示，封面图为主）
└── 最新文章（克制的列表或轻量卡片）
```

首屏推荐文案：

> 把路过的光、按下的快门和零碎的想法，收进这个小口袋。

### 3.4 摄影与相册的组织方式

- **年份是相册的主分类。**
- 摄影页按照年份展示，例如 `2026`、`2025`、`2024`。
- 每个年份内展示该年创建或拍摄的相册封面。
- 地点、主题等内容可以作为相册辅助标签显示，但不作为主导航。
- 相册列表以大封面、标题和极简辅助信息为主。

### 3.5 文章的组织方式

- 不建立文章分类。
- 文章仅使用标签组织，例如：`随笔`、`旅行`、`摄影`、`建站`。
- 文章列表按发布时间倒序。
- 标签页展示同一标签下的文章。
- 归档页按年份汇总文章。

## 4. 内容模型与后台操作

### 4.1 文章

文章使用 WordPress 默认区块编辑器（Gutenberg）创作，不使用 Markdown 编辑器。

#### 必填内容

- 标题
- 正文

#### 建议填写

- 标签
- 封面图片

#### 可选内容

- 摘要
- 定时发布时间

#### 自动提供

- 草稿保存
- 预览
- 发布日期
- 文章链接
- 修订历史

### 4.2 相册

相册是本站重要的独立内容类型，不与普通文章混用。

#### 创建相册时的最小必填项

- 标题
- 年份
- 照片列表

#### 可选项

- 相册封面图
- 拍摄日期
- 地点
- 简介
- 标签

相册封面可直接从该相册已上传的照片中选择。

### 4.3 照片

单张照片默认以“只上传文件即可展示”为原则。

#### 批量上传默认流程

```text
新建相册
→ 填写标题和年份
→ 一次选择多张照片上传
→ 照片自动归入当前相册
→ 调整顺序
→ 选择封面
→ 发布
```

不强制填写以下字段：

- 照片标题
- 照片说明
- 拍摄日期
- 地点
- 器材
- 替代文本

如以后需要精细整理，可单独对任意照片补充上述信息。

#### 前台显示规则

- 未填写图片说明：只展示图片，不显示空的信息栏。
- 已填写图片说明：在灯箱或图片详情区域显示。
- 未填写日期、地点等：不显示“未知”占位文字。
- 默认按相册内的排序展示；后台应支持拖拽调整顺序。

### 4.4 图片处理策略

- WordPress 媒体库保留上传时的原始展示文件。
- WordPress 自动生成缩略图、中等尺寸和大尺寸版本。
- 相册网格优先使用中等尺寸图片。
- 灯箱查看时使用大尺寸图片。
- 不向访客提供原始文件下载入口。
- 上传到网站的图片应是适合公开展示的导出图，不应是相机 RAW 或唯一的最高分辨率原片。

#### 原图保存原则

- 相机 RAW、未压缩原片和最高分辨率母版优先保存于 Windows 本地电脑。
- 原图至少保留一份额外离线备份，例如移动硬盘。
- 服务器只保存需要公开展示的版本；不将 RAW 挂载给公开网站。

## 5. 视觉与交互规范

### 5.1 视觉关键词

```text
蓝色 | 明亮 | 大图 | 留白 | 极简 | 温和 | 轻快 | 私人摄影册
```

“哆啦D梦的口袋”借用的是明亮、友好、蓝色、轻快的情绪，不复制任何动画角色、官方 Logo、人物形象、场景素材或受版权保护的美术元素。

### 5.2 色彩方向

- 主色：清澈、明亮的蓝色。
- 背景：暖白或近白色，避免冷灰大面积铺底。
- 点缀：少量红色与金黄色，用于按钮、提示、小型强调元素。
- 正文：深灰或接近黑色，以保证阅读对比度。

### 5.3 排版与布局

- 大图优先，页面保留充分呼吸空间。
- 使用简洁、圆润、易读的中文字体风格；不使用夸张卡通字体。
- 图片可使用适度圆角，避免厚重阴影和密集边框。
- 文章正文宽度保持克制，适合长文阅读。
- 相册详情以图片网格与留白为主体，文字说明保持简短。
- 适配桌面、平板与手机；移动端优先保证图片浏览顺畅。

### 5.4 动效原则

- 仅使用轻量、功能性动效：淡入、悬停反馈、灯箱切换、导航展开。
- 动效应短促、克制，不妨碍阅读或观看图片。
- 不使用夸张的页面转场、自动轮播或大量浮动装饰。

### 5.5 页脚规则

- 不建立传统多栏目页脚。
- 允许保留极简的底部版权或备案信息；如无必要，不展示额外链接、社交入口或友情链接。

## 7. 数据存储、备份与迁移

### 7.1 数据分别存在哪里

| 数据 | 存放位置 | 说明 |
| --- | --- | --- |
| WordPress、主题、插件代码 | Docker 镜像或项目文件 | 可由 Git 与 Docker 构建恢复。 |
| 文章、相册、标签、用户、设置 | MariaDB 数据目录 | 应定期导出 SQL 备份。 |
| 上传的展示图片和缩略图 | WordPress `uploads` 持久化目录 | 必须单独备份；真实照片文件在此处。 |
| HTTPS 证书与 Caddy 状态 | Caddy 持久化目录 | 迁移时可选择带走，也可由 Caddy 在新服务器自动重新签发。 |
| 原图与 RAW | Windows 本地及离线备份 | 不依赖服务器，不对外提供。 |

### 7.2 Docker 的角色

Docker Compose 用于保证 Caddy、WordPress、MariaDB 的运行环境可以复现。

Docker **不是**唯一的数据备份方案：

- 容器镜像带走程序和依赖；
- 数据库备份带走文章、相册、配置；
- `uploads` 备份带走所有公开照片与缩略图；
- 三者缺一不可。

### 7.3 网站迁移包

从一台服务器迁移到另一台服务器时，至少带走：

```text
site-migration-backup/
├── project/                 # Docker Compose、Caddy 配置、主题、脚本与文档
├── database.sql             # MariaDB 导出的数据库内容
├── uploads.tar.zst          # 所有 WordPress 上传照片及生成尺寸
└── env-backup/              # 非敏感配置说明；敏感密码通过安全方式单独保存
```

迁移流程原则：

1. 在旧服务器停止写入或进入维护状态，确保数据一致。
2. 导出 MariaDB 数据库。
3. 打包 `uploads` 目录。
4. 保存 Docker Compose、Caddy 配置、自定义主题与必要插件。
5. 在新服务器安装 Docker 与 Docker Compose。
6. 恢复项目配置、数据库和 `uploads` 目录。
7. 启动服务。
8. 将域名 DNS 指向新服务器。
9. 由 Caddy 自动申请新服务器上的 HTTPS 证书。
10. 验证文章、相册、图片和后台登录后再关闭旧服务器。

### 7.4 备份原则

由于不使用 COS 或外部对象存储，服务器本地磁盘是网站公开图片的唯一在线来源。因此必须主动备份：

- 定期导出 MariaDB 数据库。
- 定期打包 WordPress `uploads` 目录。
- 将备份下载到 Windows 本地或离线硬盘。
- 保留至少一份不在同一台服务器上的备份。
- 上传、删除大量图片或更新主题前，先创建一次手动备份。

## 8. 安全与运维基线

### 8.1 基础安全要求

- WordPress 后台使用强密码，后台账号仅供站点所有者使用。
- 不创建不必要的管理员账号。
- 保持 WordPress 核心、主题、插件和 Docker 镜像更新。
- 仅安装必要且可信的插件；避免为了小功能堆叠插件。
- 不安装来路不明的盗版主题、插件或代码包。
- 数据库密码与 WordPress 密钥存放在不提交至 Git 的环境变量文件中。
- 图片目录和数据库目录使用 Docker 持久化挂载，避免重建容器时丢失数据。
- 原图、RAW 与服务器备份文件不通过公开静态路径暴露。

### 8.2 HTTPS

- Caddy 负责 HTTPS 证书自动申请与续期。
- 服务器需允许公网访问 `80` 和 `443` 端口。
- 域名 DNS 必须解析到服务器公网 IP。
- 不使用 Caddy 时才需要另配 Nginx 与 Certbot；本项目选择 Caddy，因此不部署 Certbot。

### 8.3 插件控制原则

第一版优先使用 WordPress 核心功能和自定义主题完成需求。插件只在以下情况引入：

- 相册需要稳定的批量上传、拖拽排序或灯箱能力，而主题自身不适合实现；
- 安全、备份、SEO 等基础能力确实需要插件；
- 选定插件需有持续维护、良好口碑和明确兼容性。

插件清单应在实际实施阶段单独记录，避免文档中预先锁定未验证的插件。

## 9. 开发实施范围

### 9.1 自定义主题需要实现

- 全局蓝色、明亮、留白、极简视觉系统。
- 主导航与移动端导航。
- 首页首屏、最新相册和最新文章区块。
- 按年份展示相册的摄影页。
- 相册详情页：照片网格、灯箱、可选照片说明。
- 文章列表与文章详情页。
- 文章标签页。
- 按年份汇总文章和相册的归档页。
- 仅含简短文字的关于页。
- 404 页面。
- 响应式图片与移动端布局。

### 9.2 相册后台需要实现或配置

- 独立的“相册”内容类型。
- 必填标题和年份字段。
- 可选简介、地点、日期、标签和封面字段。
- 一个相册关联多张图片。
- 批量上传后自动关联到当前相册。
- 照片拖拽排序。
- 从相册照片中选择封面。
- 单张照片可选说明；未填说明时前台不显示空区域。

### 9.3 验收标准

项目第一版满足以下条件才算完成：

- 可以通过 `https://域名` 打开前台网站，证书正常。
- 可以通过 `https://域名/wp-admin` 登录后台。
- 可以创建并发布带标签的文章。
- 可以创建相册，填写标题和年份后批量上传照片并直接发布。
- 相册在摄影页按年份正确出现。
- 相册照片在桌面与移动端正常展示，点击可以灯箱浏览。
- 不填写照片信息时，前台不会出现空白元数据占位。
- 首页突出精选照片和最新相册，文章作为次级入口。
- 前台不存在评论、联系表单、社交链接、友情链接、用户注册和器材页。
- 可以完整备份数据库和上传图片，并在另一台服务器恢复。

## 10. 后续实施顺序

本项目的内容与架构已确定。实际开发按以下顺序推进：

1. 初始化 Docker Compose 环境：Caddy、WordPress、MariaDB 与持久化目录。
2. 完成域名、HTTPS、后台登录与数据库连通性。
3. 创建自定义主题基础结构和全局视觉系统。
4. 实现首页、文章、标签、归档、关于和 404 页面。
